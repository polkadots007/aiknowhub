# Architecture Decisions

## Why Zustand?

- Lightweight
- Minimal boilerplate
- Suitable for application size

## Why Supabase?

- Built-in auth
- PostgreSQL
- RLS support
- Stored Procedures
- Built-in SQL Editor and Schema Visualizer to validated changes

## Why Node backend?

- Centralized authorization
- Secure AI orchestration
- Easier security enforcement

## Why Tailwind?

- Easier css orchestration
- No overhead of maintaining separate css files
